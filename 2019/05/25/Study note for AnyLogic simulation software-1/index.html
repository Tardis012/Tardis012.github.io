工具栏学习-Agent palette

The Agent palette overview

The Agent palette contains general elements used for defining dynamics of the model, its structure and data.

Agent

有3种选择：建立Agent群、建立单个Agent、建立Agent type。

建立Agent群：

  1. 可以使用参数作为Agent群中Agent的个数。运行中如果需要增删Agent群中Agent的个数，修改该参数并不会调整，而是需要使用更改处理程序来处理Agent的创建和销毁。
  2. 建立Agent群后，可以用编号索引到单个Agent，从0 to people.size() – 1。

Parameter

参数和变量的区别：(1) 参数通常用于静态描述对象。参数通常是单次仿真中的常量，只有在需要调整模型行为时才会更改参数。(2) 变量表示模型状态，在仿真过程中可能会发生变化。

General properties

  1. 类型定义

可以使用“Type”按钮组中的相应选项选择最常用的类型之一（int，double，boolean，String等）。如果需要定义其他Java类的参数，请选择“Other”选项，然后在右侧的编辑框中键入所需的类名。

You can define parameters of your own classes, defined elsewhere in your model.

If System dynamics array is enabled, type is automatically defined as HyperArray.

So long as parameters are frequently used in system dynamics models, and sometimes these models operate with arrays, AnyLogic supports creating parameters of array type.

If you want to make the parameter a hyperarray used in system dynamics, select the System dynamics array check box, then go to the Dimensions table and select the dimensions of this array. To define default values of array elements, click the Edit… button and initialize the array as described here.

  1. 单位定义

可以使用“Unit”按钮组中的相应选项定义参数的单位。

  1. 默认值设定

在”Default value”编辑框中指定参数的默认值。之后可以使用set_parameterName（ new value ）函数在仿真期间更改参数的值。如果未指定默认值，则应用Java规则，例如，double类型的参数设置为0，boolean类型的参数为false。

Event

事件是在模型中安排某些操作的最简单方法。因此，事件通常用于模拟延迟和超时。有时您使用状态图中的定时转换执行相同操作，但事件可能更有效。

General properties

There are three types of events:

  • Timeout triggered event. It is used when you need to schedule an action at some particular moment of time (or some particular date).The event occurs exactly in timeout time after it is started. Timeout triggered event has even more features: you can specify that it expires either once or cyclically, or is fully controlled by the user. 如果为该类事件，可以设置3种不同的模式。还有一系列的设置等……
  • Condition triggered event. It is used when you want to monitor a certain condition and execute an action when this condition becomes true.
  • Rate triggered event. It is used to model a stream of independent events (Poisson stream). It is frequently needed to model arrivals: e.g. customer arrivals in queueing systems, transaction arrivals in server-based network models, etc.

Dynamic Event

动态事件和事件之间的区别在于动态事件在执行操作后会自行删除，而用户控制的事件会存活并可以重新启动。

动态事件的主要好处是可以在模型中同时调度同一动态事件的多个实例。动态事件的另一个特性是能够使用某些特定数据初始化动态事件的每个实例。在计划动态事件的新实例时，您可以使用事件的参数传递所需的数据。可以在动态事件的操作中访问这些参数。

但是，动态事件不能直接用作事件，因此我们建议仅在以下情况下使用它们：

  • When you expect several events, performing similar actions, to be scheduled at the same time in your model.
  • When the action of your dynamic event depends on some specific information.

Function

General properties

函数：仅操作 or 有返回值（设定返回值类型）。

Arguments

在这里可以为函数定义一些可选参数，以便传递函数计算所需的一些额外输入数据。

Function body

在这里输入函数体。如果函数返回一些值，则最后一行应该是返回表达式。

使用Java编写函数。

Advanced

Static – If selected, the function is static. A static function does not need creating any instances of the agent type, in whose code it is defined. Static function defined in the MyClass agent type can be accessed from any place of the model as MyClass.function().

Access – The access type of the function. There are four access types:

  • public - the function can be accessed from anywhere
  • private – the function can be accessed from this agent class only
  • protected - the function can be accessed from this agent class and its subclasses
  • default - the function can be accessed anywhere within this model.

System dynamics units – If selected, you will be able to specify measurement units for the value returned by this function in the edit box to the right. Having specified units for elements of your model, you may perform unit checking to find out dimension inconsistencies in the model.

函数的调用

默认情况下，可以从模型的任何位置调用函数。您可以通过更改函数的访问类型来限制函数的访问范围。

调用函数请写入函数名称，后跟括号。

如果函数有参数，则需要在括号内传递由逗号分隔的参数值。必须按照函数属性的Arguments部分的函数参数表中定义的顺序提供这些值。

Table Function

AnyLogic根据给定的数据和选择的插值类型构建表函数。可能需要表函数来定义复杂的非线性关系。

General properties

Interpolation – Specifies how the table function is interpolated:

  • None – no interpolation applied.
  • Step – step interpolation.
  • Linear – linear interpolation.
  • Spline – 4th order spline interpolation.
  • Approximation – The resulting function is a polynomial of Order you specify in the edit-box on the right, formed in order the sum of the root-mean-square error in the points is minimal.

Out of range – Specifies how the table function’s behavior in infeasible area (when its argument is out of range):

  • Error – runtime error is raised.
  • Nearest – the function is extrapolated by using the nearest valid argument.
  • Repeating – the function is extrapolated by making it periodic with the function range as a period.
  • Custom – the function returns a custom value, defined in the Value field to the right.
  • Extrapolate – the function is extrapolated outside the range in accordance to the interpolation type.

Schedule

AnyLogic provides users with a Schedule - a special element allowing to define how some value changes in time according to the defined (optionally cyclic) pattern.

Schedule is frequently used to define:
•Timetable for a pool of resources defined with the ResourcePool.
•Agent generation times, or time pattern of agent arrival rate in the block Source.
•Pedestrian generation times, or time pattern (e.g. daily) of pedestrian arrival rate in the block PedSource.

Moreover, you can use schedule in none AnyLogic objects but work with it programmatically, obtaining its value using the corresponding API.

Port

端口在消息传递机制中起着核心作用。端口是双向的，消息通过端口发送和接收。

Connector

要建立对象间的交互，您需要将Agent的有效接口元素与Connector连接起来。Connector是连接两个Port或两个variable的一条线。

连接两个端口意味着消息将在它们之间传递。

连接变量意味着它们在任何时刻都有相同的值（一个变量的变化将立即传播到另一个变量，声明为外部变量）。

Link to Agents

Statechart

用于绘制状态图的组件，包含Statechart Entry Point、State Transition、Initial State Pointer、Branch、History State、Final State。

使用笔记

  1. 对Agent指定可数的状态、状态之间的变迁关系、变迁条件。

  2. 向主函数和Agent里添加参数，可以设定默认值，后续可以用来作为判断条件或被更新。

  3. 主函数和Agent里都可以设置以下动作。在主函数里设置启动时Agent的初始动作。

  1. 在主函数里设置Agent群所处的空间环境。

  2. 在主函数里设置Agent状态统计函数。
